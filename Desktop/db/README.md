# Database Practice Simulation

## Objective

The objective of this practice was to create and configure a database environment, design the database structure, and perform basic data management operations.

## Steps Performed

1. Set up the database environment using Docker Compose.
2. Started the database container and verified that it was running correctly.
3. Connected to the database using a database management tool.
4. Created a new database for the project.
5. Designed the database schema based on the given requirements.
6. Created the necessary tables with their corresponding primary keys and foreign keys.
7. Defined the appropriate data types and constraints for each column.
8. Imported sample data from CSV files into the database.
9. Verified that the data was inserted correctly by running SQL queries.
10. Executed basic SQL operations such as:

    * SELECT
    * INSERT
    * UPDATE
    * DELETE
11. Validated the relationships between tables.
12. Tested the database by querying and reviewing the stored information.
13. Fixed import or data format issues when necessary.
14. Confirmed that the database was working correctly and all required information was available.

## Result

The database environment was successfully configured, the data was imported correctly, and the SQL operations were executed as expected. The final database was ready for testing and future development.
